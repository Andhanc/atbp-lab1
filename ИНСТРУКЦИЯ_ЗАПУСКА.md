# Пошаговая инструкция: запуск проекта и сдача работы (ЛР №4, вариант 14)

Соответствует ТЗ из `tz.txt`: этапы 2.2 (п. 1–5) и требования варианта 14.

---

## Шаг 1. Установка зависимостей

В корне проекта выполнить:

```bash
npm install
```

Убедиться, что установлены (в т.ч. в `devDependencies`):
- `@cucumber/cucumber`
- `supertest`
- `express`

**Что показать при сдаче:** можно скрин/вывод `npm list @cucumber/cucumber supertest express` или фрагмент `package.json` с секциями `dependencies` и `devDependencies`.

---

## Шаг 2. Запуск BDD-тестов (основная проверка по ТЗ)

Команда из ТЗ (п. 5): *«Запустить тесты через консоль командой npm run bdd»*.

```bash
npm run bdd
```

**Ожидаемый результат в консоли:**
- Сводка в формате `summary` (какие сценарии прошли/не прошли).
- Строка вида: **`8 scenarios (8 passed)`** и **`47 steps (47 passed)`**.
- В конце: **`0m00.xxx s`** (время выполнения).
- Сообщений об ошибках быть не должно.

**Что показать при сдаче:** скриншот или копия вывода консоли после `npm run bdd`, где видно, что все сценарии и шаги прошли (все зелёные / passed).

---

## Шаг 3. HTML-отчёт (требование ТЗ п. 5)

ТЗ: *«Сгенерировать развернутый HTML-отчет (--format html:report.html) для визуализации покрытия тестами»*.

Отчёт создаётся **той же** командой `npm run bdd` (в `package.json` уже указано `--format html:report.html`).

После запуска тестов:

1. В корне проекта появится файл **`report.html`**.
2. Открыть его в браузере (двойной клик или `open report.html` / `xdg-open report.html`).

**Что проверить в отчёте:**
- Все сценарии помечены **зелёным** (PASSED).
- Видны названия сценариев на русском и шаги (Дано/Когда/Тогда).
- Есть раздел с «Проверка разных текстов на процент спама» (Scenario Outline с таблицей Примеры).

**Что показать при сдаче:** скриншот открытого в браузере `report.html`, где видно зелёные сценарии и что все тесты пройдены.

---

## Шаг 4. (Опционально) Запуск сервера для ручной проверки API

ТЗ не требует обязательно запускать сервер вручную, но для демонстрации API можно:

```bash
npm start
```

Сервер слушает порт 3000. Проверка в браузере или Postman:

- **GET** `http://localhost:3000/api/status` — ответ `{ "status": "online", ... }`.
- **GET** `http://localhost:3000/api/config/spam-words` — список запрещённых слов.
- **POST** `http://localhost:3000/api/spam/check` с телом `{ "text": "купить бесплатно" }` — ответ с `spamPercentage`, `isSpam`.

**Что показать при сдаче:** по желанию — скрин вызова API (браузер/Postman), не обязательно для зачёта по п. 5 ТЗ.

## Краткий чек-лист соответствия ТЗ (вариант 14)

| Требование ТЗ | Где проверить |
|---------------|----------------|

| HTML-отчёт `report.html`, все сценарии зелёные | Шаг 3 |

## Минимальный набор для сдачи

1. Выполнить в терминале: **`npm run bdd`**.
2. Сделать **скрин вывода консоли**, где видно: все сценарии и шаги passed.
3. Открыть **`report.html`** в браузере и сделать **скрин отчёта** с зелёными сценариями.

Этого достаточно, чтобы показать соответствие п. 5 ТЗ и готовность работы к зачёту.
