Лабораторная работа №6
Настройка CI/CD для комплексных автоматизированных тестов

Цель работы: Научиться настраивать непрерывную интеграцию и доставку (CI/CD) с помощью GitHub Actions для автоматического запуска всех видов тестов (модульных, API, BDD, UI) при каждом изменении кода, а также освоить практики генерации отчётов и управления артефактами.

Основные задачи:
Изучить основы CI/CD: концепции непрерывной интеграции, непрерывной доставки и их роль в современной разработке ПО.
Освоить платформу GitHub Actions: структуру workflow-файлов, триггеры, джобы, шаги и экосистему готовых actions.
Научиться настраивать окружение для запуска тестов в виртуальной среде: установка зависимостей, запуск сервера, выполнение тестов.
Приобрести практические навыки организации параллельного выполнения тестов для ускорения обратной связи.
Овладеть техниками публикации отчётов о тестировании (HTML-отчёты Playwright, отчёты Cucumber) как артефактов сборки.
Настроить уведомления о результатах прогона тестов (в Telegram, Slack или по email).

2.1 Теоретические сведения
Пайплайн (Pipeline) — это автоматизированная последовательность этапов, которые проходит код от момента его написания до развертывания на сервере. В контексте CI/CD (Continuous Integration / Continuous Delivery) пайплайн автоматически запускает сборку, тестирование и деплой при каждом изменении в репозитории.
Пайплайн используется для следующий действий:
Автоматически запускаться при каждом пуше в репозиторий
Запускать все три вида тестов: Jest (модульные), Cucumber (API), Playwright (UI)
Генерировать Allure-отчёт и публиковать его на GitHub Pages
Отправлять уведомление в Telegram о результате
Кэшировать зависимости для ускорения работы 
Перед написанием пайплайна убедитесь, что выполнены следующие шаги:
Код в репозитории: Весь проект (включая папки .github, e2e, features, __tests__, public и файл server.js) должен быть запушен в репозиторий на GitHub.
Структура папок: GitHub Actions автоматически ищет файлы конфигурации в директории .github/workflows/. Создайте эту папку в корне вашего репозитория, если её ещё нет.
Настройка секретов: Если для деплоя или отправки уведомлений требуются токены (например, для Telegram или GitHub Packages), их необходимо добавить в настройках репозитория: Settings → Secrets and variables → Actions. В примерах ниже используется secrets.GITHUB_TOKEN (он предоставляется автоматически).
Минимальная конфигурация. Для разработки CI-паплайна необходимо в папке  .github/workflows создать файл ci.yml со следующим содержимым:
 
name: CI Pipeline

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    
    strategy:
      matrix:
        node-version: [16.x, 18.x]
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Use Node.js ${{ matrix.node-version }}
        uses: actions/setup-node@v3
        with:
          node-version: ${{ matrix.node-version }}
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Start server
        run: |
          npm start &
          npx wait-on http://localhost:3000
      
      - name: Run unit tests
        run: npm run test:unit
      
      - name: Run API tests (Cucumber)
        run: npm run test:api
      
      - name: Run UI tests (Playwright)
        run: npm run test:ui
      
      - name: Upload test results
        if: always()
        uses: actions/upload-artifact@v3
        with:
          name: test-reports-${{ matrix.node-version }}
          path: |
            reports/
            test-results/
            playwright-report/

Обратите внимание, что в данной конфигурации используется только одна задача с именем test. Можно создать несколько задач, которые будут выполняться параллельно.
В строке

    runs-on: ubuntu-latest
описывается, что тестирование применяется на виртуальной машине с операционной системой последняя версия Ubuntu. Можно настроить альтернативы: windows-latest, macos-latest.: 
Строкой 
      - name: Install dependencies
        run: npm ci
устанавливаются зависимости из package-lock.json. 
Для запуска сервера используется:
      - name: Start server
        run: |
          npm start &
          npx wait-on http://localhost:3000
wait-on — это утилита, которая приостанавливает выполнение CI-пайплайна до тех пор, пока сервер не начнёт отвечать на запросы. Без неё тесты могут начаться раньше, чем сервер запустится, что приведёт к ложным падениям.
Загрузка результатов осуществляется в следующий строках конфигурации:
      - name: Upload test results
        if: always()
        uses: actions/upload-artifact@v3
        with:
          name: test-reports-${{ matrix.node-version }}
          path: |
            reports/
            test-results/
            playwright-report/

Отчеты сохраняются как артефакты (можно скачать после выполнения).
Проверка с помощью actionlint. actionlint — это специализированный линтер для GitHub Actions workflow-файлов, который проверяет синтаксис, типы выражений, правильность использования ключей и многое другое.
Установка для Linux:
bash <(curl https://raw.githubusercontent.com/rhysd/actionlint/main/scripts/download-actionlint.bash)

Для проверки запускаем линтер через:
# Проверить все workflow-файлы
actionlint .github/workflows/*.yml

# Если хотите проверить только ваш файл ci.yml
actionlint .github/workflows/ci.yml

Что проверяет actionlint:
Синтаксические ошибки YAML
Неправильные ключи и секции
Ошибки в выражениях ${{ }}
Несуществующие свойства в контекстах (например, github.repository.permissions)
Неверные метки раннеров
Проблемы безопасности (например, прямой вывод untrusted input в скриптах) 
Параллельные тесты. Для разбиения тестов на параллельные используются несколько джобов (например для unit-тестов):
unit-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - run: npm ci
      - name: Run Unit Tests
        run: npm run test:unit
      - name: Upload Coverage Report
        uses: actions/upload-artifact@v4
        if: always()
        with:
          name: coverage-report
          path: coverage/

Аналогичным образом добавляются отдельные джобы для интеграционных тестов и тестов интерфейса.
Генерация отчетов с помощью Allure. Allure — это универсальный фреймворк для генерации красивых отчётов, который поддерживает все три типа тестов: Jest, Cucumber и Playwright.
Установка Allure:
# Установить Allure Commandline
npm install --save-dev allure-commandline

# Установить адаптеры для тестов
npm install --save-dev allure-jest @cucumber/cucumber allure-cucumberjs
npm install --save-dev @playwright/test allure-playwright

Настраиваем Jest для Allure (jest.config.allure.js):
module.exports = {
  preset: 'babel-jest',
  testEnvironment: 'node',
  testMatch: ['**/__tests__/**/*.test.js'],
  setupFilesAfterEnv: ['allure-jest'],
  reporters: [
    'default',
    ['allure-jest', {
      resultsDir: './allure-results',
      cleanResultsDir: true,
    }]
  ],
};

Настраиваем Cucumber для Allure (cucumber.js):
module.exports = {
    default: `--require steps/**/*.js features/**/*.feature --format json:allure-results/cucumber.json`
}
Настраиваем Playwright для Allure (playwright.config.js):
const { defineConfig, devices } = require('@playwright/test');

module.exports = defineConfig({

    },

    },
    {
      name: 'webkit',

    },

Для использования Allure необходимо отредактировать файл ci.yml:

name: CI Pipeline

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

env:
  ALLURE_RESULTS_DIR: allure-results

jobs:
  # ============================================
  # JOB 1: Модульные тесты (Jest + Allure)
  # ============================================
  unit-tests:
    runs-on: ubuntu-latest
    
    strategy:
      matrix:
        node-version: [16.x, 18.x]
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Use Node.js ${{ matrix.node-version }}
        uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Run unit tests (Jest + Allure)
        run: npm run test:unit
      
      - name: Upload Jest Allure results
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: allure-results-jest-${{ matrix.node-version }}
          path: allure-results/

  # ============================================
  # JOB 2: API тесты (Cucumber + Allure)
  # ============================================
  api-tests:
    runs-on: ubuntu-latest
    
    strategy:
      matrix:
        node-version: [16.x, 18.x]
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Use Node.js ${{ matrix.node-version }}
        uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Start server
        run: |
          npm start &
          npx wait-on http://localhost:3000
      
      - name: Run API tests (Cucumber + Allure)
        run: npm run test:api
      
      - name: Upload Cucumber Allure results
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: allure-results-cucumber-${{ matrix.node-version }}
          path: allure-results/

  # ============================================
  # JOB 3: UI тесты (Playwright + Allure)
  # ============================================
  ui-tests:
    runs-on: ubuntu-latest
    
    strategy:
      matrix:
        node-version: [16.x, 18.x]
        browser: [chromium, firefox, webkit]
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Use Node.js ${{ matrix.node-version }}
        uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Install Playwright browsers
        run: npx playwright install ${{ matrix.browser }} --with-deps
      
      - name: Start server
        run: |
          npm start &
          npx wait-on http://localhost:3000
      
      - name: Run UI tests (Playwright + Allure)
        run: npm run test:ui -- --project=${{ matrix.browser }}
      
      - name: Upload Playwright Allure results
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: allure-results-playwright-${{ matrix.browser }}-node-${{ matrix.node-version }}
          path: allure-results/

  # ============================================
  # JOB 4: Генерация Allure отчёта
  # ============================================
  allure-report:
    runs-on: ubuntu-latest
    needs: [unit-tests, api-tests, ui-tests]
    if: always()
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Install dependencies
        run: npm ci
      
      - name: Download all Allure results
        uses: actions/download-artifact@v4
        with:
          path: all-allure-results/
      
      - name: Merge Allure results
        run: |
          mkdir -p merged-allure-results
          find all-allure-results -name "*.json" -exec cp {} merged-allure-results/ \;
          find all-allure-results -name "*.txt" -exec cp {} merged-allure-results/ \;
          find all-allure-results -name "*.png" -exec cp {} merged-allure-results/ \;
      
      - name: Generate Allure report
        run: |
          npx allure generate merged-allure-results --clean -o allure-report
      
      - name: Upload Allure report as artifact
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: allure-report
          path: allure-report/
      
      - name: Deploy Allure report to GitHub Pages
        if: github.ref == 'refs/heads/main'
        uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./allure-report
          destination_dir: allure-reports/${{ github.run_number }}

  # ============================================
  # JOB 5: Уведомление (опционально)
  # ============================================
  notify:
    runs-on: ubuntu-latest
    needs: [unit-tests, api-tests, ui-tests, allure-report]
    if: always()
    
    steps:
      - name: Check overall status
        run: |
          if [ "${{ needs.unit-tests.result }}" = "success" ] && \
             [ "${{ needs.api-tests.result }}" = "success" ] && \
             [ "${{ needs.ui-tests.result }}" = "success" ]; then
            echo "✅ All tests passed!"
          else
            echo "❌ Some tests failed. Check Allure report for details."
            exit 1
          fi

Для  проверки генерации отчетов запустить Allure локально:
# Запустить все тесты и сгенерировать Allure отчёт
npm run test:all

# Или по отдельности:
npm run test:unit      # Jest → allure-results/
npm run test:api       # Cucumber → allure-results/
npm run test:ui        # Playwright → allure-results/

# Сгенерировать и открыть отчёт
npm run allure:generate
npm run allure:open

# Или сразу сгенерировать и открыть
npx allure serve allure-results

Таким образом, на примере показано как настраивать тесты в рамках непрерывного тестирования, проверять созданный файл конфигурации CI-паплайна и настраивать наглядные отчеты с помощью фремворка Allure.

2.2 Этапы выполнения лабораторной работы 
1. Подготовка
Убедиться, что проект имеет структуру, аналогичную описанной в предыдущей работе (с папками __tests__, features, tests, public).
Создать публичный репозиторий на GitHub и загрузить туда весь код.
Проверить package.json на наличие скриптов для запуска всех тестов. Минимальная конфигурация:
"scripts": {
  "test:unit": "jest",
  "test:api": "cucumber-js",
  "test:ui": "npx playwright test",
  "test:all": "npm run test:unit && npm run test:api && npm run test:ui",
  "start": "node server.js"
}

2. Создание базового CI-пайплайна:
Для работы с репозиторием и созданием CI-пайплайна cоздать в корне проекта папку .github/workflows.
В вновь созданном файле .github/workflows/ci.yml записать содержимое CI-пайплайна для разработанной функции. Настроить матрицу Node.js версий (18, 20, 22) и убедиться, что проект работает на всех версиях. Добавить в матрицу разные ОС (ubuntu-latest, windows-latest, macos-latest) и обеспечить прохождение тестов на всех платформах. 
Для проверки правильности CI-пайплайна составленного установить actionlint.
Для ускорения работы разделить тесты на отдельные работы, для параллельного выполнения. 
3. Генерация и публикация отчётов
Установить Allure для красивых отчётов.
Настроить Allure для сбора информации о всех проведенных тестах.
Настроить генерацию Allure-отчётов и их публикацию на GitHub Pages.
4. Настройка уведомлений в социальные сети:
Добавить уведомления в Telegram.
5. Оптимизация и запуск в работу CI-пайплайна:
Настроить кэширование node_modules и браузеров Playwright.
Закачать полученный CI-пайплайн на github.  Проверить работу CI-пайплайна. Добавить изменения в код программы, добавить для добавленного изменения тесты, закачать на github и проверить работу CI-пайплайна. Проверить получение уведомлений в соц сети.
Продвинутый уровень.
Обернуть приложение в Docker, написать Dockerfile и настроить запуск тестов внутри контейнера в CI.
